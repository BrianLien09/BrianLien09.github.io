題目 A: Elderly on the Trail

解題重點

本題的目標是計算一個指定座標 (x, y) 的「訊號良好分數 S」。訊號的覆蓋範圍是基於「曼哈頓距離」，也就是 |x1 - x2| + |y1 - y2|。

主要邏輯如下：

  1. 讀取要計算分數的目標座標 (x, y) 以及監測器的數量 N。
  2. 遍歷 N 個監測器，讀取每一個監測器的位置 (a, b) 和訊號強度 c。
  3. 計算目標座標與當前監測器之間的曼哈頓距離 dist。
  4. 如果 dist <= c，表示該監測器的訊號有覆蓋到目標座標，將計數器 count 加一。
  5. 所有監測器都檢查完畢後，判斷 count 的值。如果 count >= 3，則分數 S 就是 count；否則，分數 S 為 0。
  6. 輸出分數 S。

題目 B: Jimmy’s LeetCode Penalty Tracker

解題重點

本題要求計算一個累進的罰款總額。規則是：每連續一天不解題，罰款金額就會增加。

主要邏輯如下：

  1. 使用一個變數 price 來累計總罰款。
  2. 使用一個變數 day 來追蹤「連續未解題」的天數，這個變數也代表罰款的倍率。
  3. 遍歷每一天的活動紀錄（1 代表解題，0 代表未解題）。
  4. 如果當天活動為 1 (解題)，則將連續未解題天數 day 重設為 1。
  5. 如果當天活動為 0 (未解題)，則將當天的罰款 day * 10 加入總罰款 price，然後將 day 加一，為下一天的可能罰款做準備。
  6. 遍歷完所有天數後，輸出總罰款 price。

題目 D: CheatCode Decoder I

解題重點

這是一個簡單的二進位解碼問題。題目給定一個由 0 和 1 組成的字串，需要將其每兩個字元一組進行解碼。

主要邏輯如下：

  1. 讀取包含二進位碼的整個字串 s。
  2. 使用 for 迴圈，每次遞增 i 兩次 (i += 2)，這樣可以一次處理兩個字元。
  3. 在迴圈中，取出當前的兩個字元 s.charAt(i) 和 s.charAt(i + 1)。
  4. 使用 if-else if 結構判斷這兩個字元的組合：
    • 00 -> A
    • 01 -> B
    • 10 -> C
    • 11 -> D
  5. 直接印出對應的字母，題目要求輸出間不能有空格。

題目 E: Gift Selector

解題重點

本題要求判斷一個可能非常大的數字 n 是否為 330 的倍數。因為 n 的位數可能高達 10,000 位，所以不能直接用 long 或 int 來儲存。

主要邏輯是利用模運算的性質來處理大數：(A * 10 + B) % M = ((A % M) * 10 + B) % M。

  1. 將輸入的數字 n 當作字串 s 讀取。
  2. 初始化一個變數 mod 為 0，用來儲存對 330 取模的餘數。
  3. 遍歷字串 s 的每一個字元（即數字的每一位）。
  4. 在迴圈中，更新 mod 的值：mod = (mod * 10 + digit) % 330，其中 digit 是當前字元轉換成的數字。
  5. 遍歷完所有位數後，如果最終 mod 的值為 0，則表示原數可以被 330 整除，輸出 “nice gift”。
  6. 否則，輸出 “skip”。

題目 G: Shanghai’s tap water comes from the sea

解題重點

本題要求判斷一個給定的字串是否為「迴文」（Palindrome）。迴文是指一個字串從前面讀和從後面讀都是一樣的。

主要邏輯如下：

  1. 讀取輸入的字串 s。
  2. 建立一個新的空字串（或使用 StringBuilder）down，用來儲存 s 的反轉版本。
  3. 從 s 的最後一個字元開始，反向遍歷到第一個字元。
  4. 將遍歷到的每個字元依序附加到 down 的末尾。
  5. 遍歷結束後，down 就會是 s 的完整反轉字串。
  6. 使用 .equals() 方法比較原字串 s 和反轉後的字串 down 是否完全相同。
  7. 如果相同，則為迴文，輸出 “YES”；否則輸出 “NO”。

題目 H: Sobriety Test I

解題重點

本題要求根據答題結果（O 為答對，X 為答錯）判斷學生是否 “drunk”。規則是：如果答對的題目數量少於答錯的數量，則判定為 “drunk”。最後計算出所有 “drunk” 的學生總數。

主要邏輯如下：

  1. 讀取學生總數 N。
  2. 初始化一個計數器 total 為 0，用來記錄 “drunk” 的學生人數。
  3. 迴圈 N 次，處理每個學生的答題記錄。
  4. 在迴圈中，讀取代表答題結果的字串 s。
  5. 為每個學生初始化兩個計數器 correct 和 incorrect。
  6. 遍歷字串 s，計算 O 和 X 的數量。
  7. 比較 correct 和 incorrect 的數量。如果 correct < incorrect，則該學生為 “drunk”，將 total 計數器加一。
  8. 所有學生都判斷完畢後，輸出 total 的值。
